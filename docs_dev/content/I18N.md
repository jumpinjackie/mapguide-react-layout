# i18n String Bundle Helper Tool

The viewer ships with a helper tool (`tools/i18n-tools.ts`) that assists with creating and maintaining
locale-specific string bundle translations.

## Background

The English string bundle (`src/strings/en.ts`) is the authoritative source of all translatable strings.
Non-English translations are stored as JSON files in `viewer/strings/` (e.g. `viewer/strings/de.json`).

The helper tool uses the English bundle as a baseline to detect gaps in any target locale bundle and
optionally auto-translate any missing keys using an external translation engine.

## npm scripts

Two convenience scripts are provided in `package.json`:

| Script | Description |
|--------|-------------|
| `yarn i18n:check` | Check all locale bundles (or a specific one) for missing keys |
| `yarn i18n:translate` | Check and auto-translate missing keys using the configured engine |

## Checking for missing keys

Run the following to check **all** locale bundles in `viewer/strings/` for missing keys:

```bash
yarn i18n:check
```

To check only a specific locale (e.g. German):

```bash
yarn i18n:check -- --locale de
```

The tool prints each missing key together with its English source value, for example:

```
📦 Locale: de
  ⚠️  Found 3 missing key(s) in 'de' bundle:
     - ENABLE_SELECT_DRAGPAN: "Enable pan dragging for select tool"
     - CREATE_VECTOR_LAYER: "Vector Layer"
     - EXTERNAL_LAYERS: "External Layers"
```

When a bundle is fully up-to-date the tool confirms:

```
📦 Locale: de
  ✅ No missing keys in 'de' bundle
```

## Auto-translating missing keys

> [!WARNING]
> Auto-translated strings are a starting point only. Machine translations, especially for
> domain-specific or technical strings, should always be reviewed and corrected by a native speaker
> before being committed.

Pass `--translate` (included automatically by `yarn i18n:translate`) to have the tool auto-translate
any missing keys and `--save` to persist the results back to the JSON bundle file:

```bash
# Translate and save all missing German keys
yarn i18n:translate -- --locale de --save
```

Without `--save` the translations are shown in the console but not written to disk — useful for
previewing results before committing:

```bash
yarn i18n:translate -- --locale de
```

## Translation engines

The tool integrates with the [`translate`](https://www.npmjs.com/package/translate) npm package.
The following engines are supported:

| Engine | `--engine` value | API key required? | Notes |
|--------|-----------------|-------------------|-------|
| Google Translate (free tier) | `google` *(default)* | No | Rate-limited; suitable for occasional use |
| LibreTranslate | `libre` | No *(self-hosted)* | Self-hosted open-source engine |
| DeepL | `deepl` | Yes | High-quality translations; requires a paid/free API key |
| Yandex Translate | `yandex` | Yes | Requires a Yandex API key |

### Google (default)

No configuration required for occasional use:

```bash
yarn i18n:translate -- --locale fr --save
```

### LibreTranslate

Requires a running [LibreTranslate](https://libretranslate.com/) instance. Pass the base URL with
`--libre-url`:

```bash
yarn i18n:translate -- --locale fr --engine libre --libre-url http://localhost:5000 --save
```

### DeepL

Obtain a free or paid API key from [deepl.com](https://www.deepl.com/pro-api) and pass it with
`--key`:

```bash
yarn i18n:translate -- --locale fr --engine deepl --key YOUR_DEEPL_KEY --save
```

### Yandex

Obtain an API key from the [Yandex Cloud console](https://cloud.yandex.com/en/services/translate) and
pass it with `--key`:

```bash
yarn i18n:translate -- --locale fr --engine yandex --key YOUR_YANDEX_KEY --save
```

## Adding a new locale

1. Create an empty JSON file at `viewer/strings/{locale}.json`, for example `viewer/strings/fr.json`:

   ```json
   {}
   ```

2. Run the translate script to populate it:

   ```bash
   yarn i18n:translate -- --locale fr --save
   ```

3. Review each translated string and correct any that are inaccurate.

4. Register the bundle in the viewer at runtime by calling `Registry.registerStringBundle`:

   ```javascript
   // In your viewer bootstrap script
   fetch("strings/fr.json")
       .then(r => r.json())
       .then(bundle => MapGuide.Registry.registerStringBundle("fr", bundle));
   ```

   Alternatively, load the bundle via the Application Definition / Web Layout locale setting.

## Full CLI reference

```
i18n String Bundle Helper Tool

Usage:
  tsx tools/i18n-tools.ts [options]

Options:
  --locale <code>    Target locale code (e.g. de, fr, ja).
                     If omitted, all locale files in viewer/strings/ are checked.
  --translate        Auto-translate missing keys using the configured translation engine.
  --engine <name>    Translation engine to use (default: google).
                     Supported: google, yandex, libre, deepl
  --key <api-key>    API key for the translation engine (required for yandex, deepl).
  --libre-url <url>  Base URL of a LibreTranslate instance (required for the libre engine).
  --save             Write auto-translated keys back to the bundle JSON file.
  -h, --help         Show this help text.
```
